waiterbrain

voice-to-order AI for restaurants. servers wear a lapel mic and belt pack. the system listens, transcribes, parses orders, and pushes them to Toast POS automatically. no manual re-entry. all processing runs locally on a Mac Mini. no cloud.

how it works

  1. server speaks order naturally into lapel mic
  2. belt pack captures audio, finds position via UWB, reads heading via IMU
  3. audio compressed with Opus codec, streamed over 915MHz radio to Mac Mini
  4. Mac Mini runs Whisper for transcription, local LLM for order parsing
  5. UWB position matched to nearest table from restaurant floor map
  6. structured order JSON pushed to Toast POS at the correct table
  7. staff approves, edits, or discards the draft on screen

software pipeline

  1. audio capture: INMP441 mic on ESP32-S3, 16kHz mono
  2. Opus compression: 256kbps raw down to 24kbps
  3. 915MHz streaming: FSK mode, TDMA for multiple servers
  4. Whisper transcription: mlx-whisper large-v3-turbo on Mac Mini
  5. LLM order parsing: local 7-14B model, structured JSON output
  6. menu validation: checks items, modifiers, rules against restaurant menu
  7. table association: UWB position matched to floor map
  8. Toast POS push: order appears as draft for staff confirmation

privacy

all processing local. no cloud dependency. no conversation data stored permanently. no audio leaves the building.

product principles

. invisible: works in the background, servers don't change behavior

. assistive: supports staff, does not replace them

. private: everything runs on-premise

. accurate: reduces missed items, modifier errors, re-entry mistakes

. learnable: improves from staff edits and discards over time

. practical: full shift battery, overnight charging, minimal setup
